2.4 运行模式（Modem）

  • Single-channel：只提供 io，内部 CMUX 拆 AT + PPP。
  • Multi-channel：提供 at_io + data_io，不用 CMUX。自动判定：data_io != null 即 multi。

2.5 Runtime 合约（Thread / Notify / Time）

来自 runtime/thread.zig、runtime/sync/notify.zig、runtime/time.zig；comptime 注入。

  • Thread：spawn/join/detach。用途：Cellular worker 线程、CMUX pump 线程。
  • Notify：init/deinit/signal/wait/timedWait。用途：CMUX 各虚拟通道的数据到达通知。
  • Time：nowMs/sleepMs。用途：AtEngine 超时循环、tick 间隔；测试用 FakeTime。
  • Channel（runtime/channel_factory.zig）：request_queue capacity=4，response_channel capacity=1。

7. 用法示例

7.1 ESP32 UART single-channel（含 Injector）

const cellular = @import("embed").pkg.cellular;
const quectel = cellular.modem.profiles.quectel;
const CellPeripheral = cellular.Cellular(EspThread, EspNotify, EspTime, quectel, EspGpio, 1024);
const uart_io = io.fromUart(UartDriver, &uart_driver);
const injector = rt.bus.Injector(.cellular);
var cell = try CellPeripheral.init(allocator, .{ .io = uart_io, .time = board.time, .gpio = &gpio, ... }, injector, .{ .poll_interval_ms = 1000 });
try cell.start();
while (true) { const r = try rt.recv(); if (!r.ok) break; rt.dispatch(r.value); if (rt.isDirty()) { ... render ...; rt.commitFrame(); } }
cell.stop();

7.2 Linux USB multi-channel

提供 at_io + data_io，Modem 使用 multi-channel 模式，无需 CMUX。主循环同 7.1。

7.3 Test mock

方式 A：真实 Bus，主线程 rt.recv() 收事件后断言。方式 B：测试用 injector 将 payload 写入 test_channel，单测不依赖完整 Bus。

7.4 Modem-only（无状态机/Bus）

直接使用 Modem.init(...)，modem.at().send(quectel.commands.GetModuleInfo, .{}) 等，无状态机、无 event channel。
